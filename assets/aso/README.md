## Content Overview

All store listings include:

### 1. Basic Information

- App name: Stay at Scandinavia
- Short description/subtitle
- Promotional text (where applicable)

### 2. Main Description Sections

- **Hero Statement**: "Experience Authentic Scandinavian Living in Indonesia"
- **What Makes Us Special**: Unique selling points (Nordic design, premium location, privacy, smart home)
- **Premium Amenities**: Detailed breakdown by room (living, bedroom, kitchen, bathroom)
- **Perfect For**: Target audience segments (business, families, couples, digital nomads)
- **Building Facilities**: Security, parking, pool, fitness, mall access
- **Location**: Tangerang City Mall benefits
- **Hygge Philosophy**: Danish concept of comfort and well-being
- **App Features**: Platform-specific functionality
- **Book Direct Benefits**: Rate guarantee, no fees, flexible cancellation
- **Guest Reviews**: Social proof with real testimonials
- **24/7 Support**: Customer service availability
- **Eco-Friendly**: Sustainability commitment

**NOTE**: Apple App Store versions (iOS and macOS) have all emojis removed as they are not allowed in App Store Connect. Microsoft Store and Google Play versions retain emojis for better visual appeal.

### 3. Platform-Specific Elements

#### Microsoft Store

- Character limit: ~10,000 characters
- Includes contact email and website
- Focus on Windows desktop experience

#### Apple iOS App Store

- App subtitle: 30 characters max
- Promotional text: 170 characters max
- Description: 4,000 characters max
- Keywords: 100 characters max
- Includes screenshot text suggestions
- Age rating: 4+
- Categories: Travel (Primary), Lifestyle (Secondary)

#### Apple macOS App Store

- Same structure as iOS but with Mac-specific features:
  - Native macOS app performance
  - Large screen optimization
  - Touch Bar support
  - Dark Mode support
  - Keyboard shortcuts
  - iCloud sync capabilities
- System requirements: macOS 11.0+, Apple Silicon/Intel

## ASO Best Practices Applied

1. **Front-Loading Keywords**: Important keywords appear early in titles and descriptions
2. **Natural Language**: Keywords integrated naturally, not stuffed
3. **Benefit-Focused**: Emphasizes user benefits over features
4. **Visual Structure**: Uses bullet points for scannability (emojis removed from Apple versions per App Store guidelines)
5. **Social Proof**: Includes testimonials and ratings
6. **Call-to-Action**: Clear download and booking prompts
7. **Localization**: Proper cultural adaptation for Indonesian market
8. **Character Limits**: Adheres to platform-specific limits
9. **Unique Value Proposition**: Clear differentiation (Nordic design + Indonesian hospitality)
10. **Search Intent**: Addresses both accommodation and experience seekers
11. **Platform Compliance**: Apple versions comply with App Store Connect restrictions (no emojis)

## Translation Notes

## Usage Guidelines

When submitting to app stores:

1. **Copy the relevant content** from the appropriate file for your platform and language
2. **Adjust formatting** to match the store's input fields (remove markdown if needed)
3. **Verify character counts** against platform limits
4. **Prepare screenshots** using the text suggestions provided
5. **Update "What's New"** section for each version release
6. **Test all URLs** before submission (support, marketing, privacy policy)
